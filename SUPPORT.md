# Support

## Use the Right Channel

- GitHub Discussions for setup questions, usage help, architecture discussion, and ideas
- GitHub Issues for reproducible bugs and concrete feature requests
- GitHub private vulnerability reporting for security issues

Repository discussions:

- <https://github.com/Rubiss-Projects/agile-tools/discussions>

## Before Opening an Issue

- Read the README and quickstart documentation.
- Search existing issues and discussions first.
- Gather reproduction steps, versions, logs, and screenshots where relevant.
- Do not include secrets, PATs, database credentials, or sensitive customer data.

## Support Expectations

- Support is provided on a best-effort basis.
- This repository does not currently offer commercial support or formal response SLAs.
- High-quality reports with clear reproduction details are much easier to triage quickly.
